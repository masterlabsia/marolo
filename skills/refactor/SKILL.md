# Skill: Refactor

## Purpose
Refatorar codigo mantendo comportamento externo identico.

## Trigger
Use quando: "refatore isso", "melhore a estrutura", "esta muito complexo"

## Principles
1. Red -> Green -> Refactor: so refatorar com testes passando
2. Small steps: commits atomicos por transformacao
3. Rename before restructure: clareza de nomes antes de mover codigo

## Common Patterns
- Extract Function / Extract Method
- Replace Magic Number with Constant
- Decompose Conditional
- Replace Temp with Query

## Output
- Codigo refatorado com comentarios explicando cada mudanca
- Lista de transformacoes aplicadas
- Confirmacao de que testes continuam passando
